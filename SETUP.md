# Full Setup Guide
## Fault Reporting App — Firebase + Africa's Talking SMS

---

## Overview of what happens when a user submits a report

```
Flutter App
  └─► Firebase Storage    (uploads photo)
  └─► Firestore           (saves report with status: "pending")
  └─► Cloud Function      (sendFaultSms)
        └─► Africa's Talking API
              └─► SMS to maintenance team phone ✅
```

---

## STEP 1 — Create a Firebase Project

1. Go to https://console.firebase.google.com
2. Click **"Add project"** → name it `hawassa-fault-reporting`
3. Disable Google Analytics (not needed) → **Create project**
4. In the left sidebar enable these services:
   - **Firestore Database** → Create in **production mode**
   - **Storage** → Start in **production mode**
   - **Functions** → requires **Blaze (pay-as-you-go) plan** (free tier is generous)

---

## STEP 2 — Connect Flutter to Firebase (FlutterFire CLI)

```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# In your flutter_fault_app folder:
flutterfire configure --project=hawassa-fault-reporting
```

This auto-generates `lib/firebase_options.dart` — **do not edit it manually**.

---

## STEP 3 — Get Africa's Talking credentials

1. Sign up at https://africastalking.com
2. For testing: use the **Sandbox** environment (free)
3. For production: top up your account with credit
4. Go to **Settings → API Key** → copy your API key
5. Your username is the name you signed up with (e.g. `hawassa_uni`)

> **Ethiopia note:** Africa's Talking supports Ethiopian numbers (+251xxx).
> You may need to request a **Sender ID** (e.g. `HU-Faults`) from their support
> for production. In sandbox mode any sender works.

---

## STEP 4 — Set Africa's Talking credentials as Firebase config

```bash
# Install Firebase CLI if you haven't
npm install -g firebase-tools

# Login
firebase login

# Set your credentials (never hard-code these in source)
firebase functions:config:set \
  at.username="YOUR_AT_USERNAME" \
  at.api_key="YOUR_AT_API_KEY"

# Verify
firebase functions:config:get
```

---

## STEP 5 — Change the maintenance team phone number

Open `lib/services/fault_service.dart` and update this one line:

```dart
// ─── Maintenance team recipient number ───────────────────────
static const String maintenancePhoneNumber = '+251911234567';
//                                            ^^^^^^^^^^^^^^
//                    Change this to the real maintenance team number
// ─────────────────────────────────────────────────────────────
```

Use international format with `+` prefix:
- Ethiopia: `+251911234567`
- You can also make this a list to notify multiple people (ask for that feature)

---

## STEP 6 — Deploy the Cloud Function

```bash
# From inside flutter_fault_app/
cd functions
npm install
npm run build
cd ..

firebase deploy --only functions
```

You should see:
```
✔ functions[sendFaultSms]: Successful create operation.
```

---

## STEP 7 — Deploy Firestore & Storage rules

```bash
firebase deploy --only firestore:rules,storage
```

---

## STEP 8 — Run the Flutter app

```bash
flutter pub get
flutter run
```

## Viewing submitted reports

Go to https://console.firebase.google.com → your project → **Firestore Database**

You'll see a `fault_reports` collection. Each document looks like:

```json
{
  "trackingId": "FLT-A3X9K2M",
  "phoneNumber": "+251911111111",
  "description": "Broken projector in IoT Lab",
  "photoUrl": "https://storage.googleapis.com/...",
  "location": {
    "commonName": "IoT Lab Room 101",
    "latitude": 7.0621,
    "longitude": 38.4755
  },
  "status": "pending",
  "submittedAt": "2025-03-01T10:30:00.000Z"
}
```

You can manually change `status` to `in_progress` or `resolved` directly in the console.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `firebase_options.dart not found` | Run `flutterfire configure` again |
| `core/no-app` or `No Firebase App [DEFAULT]` on submit | Firebase init failed. Run `flutterfire configure`, ensure Android app is registered, then rebuild (`flutter clean && flutter pub get && flutter run`) |
| SMS not received | Check Africa's Talking dashboard → Sent Messages |
| Cloud Function error | Run `firebase functions:log` |
| Sandbox SMS not delivered | In AT sandbox, you must add the recipient number to test accounts first |
| `PERMISSION_DENIED` on Firestore | Check `firestore.rules` is deployed |
